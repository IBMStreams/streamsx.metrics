SystemStatus

The SystemStatus type specifies all attributes that are filled by the SystemMonitorSource operator

  • status

    Date and time of the monitored values. If parameter details is set to true, then this attribute contains additional information retrieved from top system command.

  • cpuUsage

    The "recent cpu usage" for the whole system.

  • memUsage

    The "recent cpu usage" for the whole system.

  • memTotal

    The total available physical memory on the system in kB.

  • memUsed

    The amount of physical memory that is used on the system in kB.

SystemStatus = tuple<rstring status, float32 cpuUsage, float32 memUsage, int32 memTotal, int32 memUsed>;

LogNotification

The LogNotification type specifies all attributes that are filled by the LogSource operator

  • notifyType

    The notification type com.ibm.streams.management.log.application.error or com.ibm.streams.management.log.application.warning.

  • domainId

    The domain ID.

  • instanceId

    The instanceId attribute specifies the unique instance id for the domain.

  • jobId

    Each job in an instances has a unique job id. See IBM Knowledge Center for more details.

  • resource

    The identifier of the resource on which the PE containing the operator is running. This is typically the host name. See IBM Knowledge Center for more details.

  • peId

    IBM Streams runs operators in processing elements (PE), which is an operating system process that includes the operators and streams that are defined in a data flow graph or subgraph of a stream processing application. Each PE has a unique id that is assigned to the peId attribute.

  • operatorName

    The name of the operator sending the log message.

  • sequence

    The notification sequence number.

  • eventTimestamp

    The notification timestamp.

  • message

    The notification message.

LogNotification = tuple<rstring notifyType, rstring domainId, rstring instanceId, int64 jobId, rstring resource, int64 peId, rstring operatorName, int64 sequence, timestamp eventTimestamp, rstring message>;
