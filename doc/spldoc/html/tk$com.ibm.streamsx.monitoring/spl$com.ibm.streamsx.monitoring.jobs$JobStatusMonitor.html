SPL File JobStatusMonitor.spl

IBMStreams streamsx.monitoring Toolkit > com.ibm.streamsx.monitoring 2.0.0 > com.ibm.streamsx.monitoring.jobs > JobStatusMonitor.spl

Content

Operators
  • JobStatusMonitor: This operator uses JobStatusSource and prevents near duplicate notifications being emitted.

Composites

composite JobStatusMonitor(output stream<JobStatusNotification> OutStream, stream<ConnectionNotification> ConnectionNotificationStream)

This operator uses JobStatusSource and prevents near duplicate notifications being emitted.

Parameters

  • applicationConfigurationName: Specifies the name of application configuration object that can contain domainId, connectionURL, user, password, and filterDocument properties. The application configuration overrides values that are specified with the corresponding parameters.
  • user: Specifies the user that is required for the JMX connection. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value.
  • password: Specifies the password that is required for the JMX connection. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value.
  • filterDocument: Specifies the either a path to a JSON-formatted document or a JSON-formatted String that specifies the domain, instance and job filters as regular expressions. Each regular expression must follow the rules that are specified for Java Pattern. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value. If parameter is not set, then all jobs in current domain and instance are monitored.
  • connectionURL: Specifies the connection URL as returned by the streamtool getjmxconnect command. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value.If not specified and the domainId parameter value equals the domain id under which this operator is running, then the operator uses the streamtool getjmxconnect command to get the value.
  • domainId: Specifies the domain id that is monitored. If no domain id is specified, the domain id under which this operator is running is used. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value.
  • sslOption: Specifies the sslOption that is required for the JMX connection. If the applicationConfigurationName parameter is specified, the application configuration can override this parameter value. If not specified and the domainId parameter value equals the domain id under which this operator is running, then the operator uses the streamtool getdomainproperty command to get the value.
  • credentials: Specifies Streaming Analytics service credentials in JSON format. Relevant for IAM authentication case only. If parameter is set, then the parameters user and password are ignored. If the applicationConfigurationName parameter is specified, the application configuration with the property credentials can override this parameter value.
  • timeOut: Specifies the number of seconds during which no duplicate of a tuple is emitted. If this parameter is not specified, the default value is 5.0 seconds (5 seconds). Identical tuples, which are separated by more than timeOut seconds, are seen on the output port.

Output Ports

  • OutStream: The notification tuples on output port 0. The schema is defined by the JobStatusNotification type.
  • ConnectionNotificationStream: The JMX connection notification tuples on output port 1. The schema is defined by the ConnectionNotification type.
