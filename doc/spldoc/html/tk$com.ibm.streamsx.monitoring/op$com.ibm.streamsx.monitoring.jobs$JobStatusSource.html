Operator JobStatusSource

IBMStreams streamsx.monitoring Toolkit > com.ibm.streamsx.monitoring 1.6.0 > com.ibm.streamsx.monitoring.jobs > JobStatusSource

The JobStatusSource operator uses the JMX API to retrieve status updates from one or more jobs, and provides status changes as tuple stream.

The operator emits tuples for the following notification types:
  • com.ibm.streams.management.job.added
  • com.ibm.streams.management.job.removed
  • com.ibm.streams.management.pe.changed

Per default, the JobStatusSource operator monitors all jobs in the current domain and instance, if filter document is not specified.

The JobStatusSource operator monitors filter-matching domains, instances, and jobs that are running while the application that uses the JobStatusSource operator, starts. Furthermore, the operator gets notifications for created and deleted instances, and submitted and cancelled jobs. Thererfore, the operator can retrieve events from filter-matching jobs that are submitted in the future.
